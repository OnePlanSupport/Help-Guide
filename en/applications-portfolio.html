Applications Portfolio

What is the Application Portfolio?

Portfolio demand can come from multiple sources including from internal applications you support, products and services you provide, and even process improvement initiatives like value streams. The application portfolio in OnePlan provides an organized list of software applications used within an organization, highlighting their status, process stage, linked business unit, cost, and benefits. This portfolio assists in assessing the impact and health of the technology portfolio, aiding the prioritization of IT projects, and aligning IT investments with business strategies. It is customizable, allowing for the display of information as needed.

Example Use Case

For example, if you are the marketing manager overseeing the introduction of a new Customer Relationship Management (CRM) system, you could use OnePlan's Application Portfolio could track the system's status, identify any workload bottlenecks or technical barriers early on, and manage the budget effectively. In case of any deviations, like the status changing to 'at risk,' you could conduct a timely investigation to ensure smooth progress. Thus, the Application portfolio serves as an essential tool for efficient project management and proactive decision-making.

Use the Applications Portfolio

For example, to view a list of  Application  plans that have been added to the  My Application Area , select the  Application Summary  view from the list drop-down. This view presents a detailed list of applications, associated Business Unit, Lifecycle Stage, functional fit, and budget.

Change the views to look at the  Applications  from a different perspective. For instance, here  Applications  are grouped by  Category .

Each  Application  plan comes with an exhaustive  Details  page where you can manage metadata like the application stages, application vision, prioritization scoring, and financials. You can also link associated projects, ideas, and epics to these plan types.

Application Board

The Application Board visualizes applications based on Business Units, Status, dependencies, and functional fit.

Configurable Views

Configurable views let you create varied application matrices. Mapping your application journey becomes straightforward with this roadmap. Group by category, color code, or edit on the go – the flexibility is immense. In this example, you can visualize the Applications Plan in a Road Map view.
